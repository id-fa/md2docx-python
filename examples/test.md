# テスト文書

## はじめに

本文書は md2word のスタイル適用をテストするためのサンプルである。標準（Normal）スタイルで本文が表示され、見出しには自動番号が付与される。

日本語と English が混在するテキストでの表示確認を行う。REST API などの技術用語も含む。

## システム概要

### アーキテクチャ

システムは以下の構成で動作する。

- フロントエンド: React + TypeScript
- バックエンド: Python / Flask
- データベース: PostgreSQL

## 詳細設計

### API 仕様

エンドポイントの一覧を以下に示す。

| メソッド | パス                 | 説明                 |
| -------- | -------------------- | -------------------- |
| GET      | /api/v1/datasets     | データセット一覧取得 |
| POST     | /api/v1/datasets     | データセット登録     |
| DELETE   | /api/v1/datasets/:id | データセット削除     |

### 認証方式

認証には **JWT トークン** を使用する。`Authorization: Bearer <token>` ヘッダで送信する。

#### トークン発行フロー

トークンの有効期限は *30分* とし、リフレッシュトークンで更新する。

1. クライアントがログイン情報を送信
2. サーバーがJWTを発行
3. クライアントがヘッダに付与してリクエスト

#### エラーハンドリング

認証エラー時は以下の JSON を返す。

```json
{
  "error": "unauthorized",
  "message": "Invalid or expired token"
}
```
